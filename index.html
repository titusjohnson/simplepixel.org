Rapid prototyping with deployd, finch.js, and knockout.js

I do a fair amount of prototyping work, often requiring a lot of the same boilerplate. This collection of code is the result of a few recreations of the same boilerplate.

Deployd has proven to be a very handy utility for socking away some data and not worrying about how it comes in and out.

KnockoutJS continues to be my favorite utility, its versitality is remarkable. I continue to be able to reduce hundreds of lines of jQuery into a few dozen lines of Knockout models, and consise template code. For this project Knockout provides the bulk of the functionality.

Finch.js provides routing, simple as that. It's a very nice piece of code, and I'm afraid to say that I haven't explored all of its functionality yet. It was quite easy to get the routing table together however.

Lastly we have Grunt and Foundation. I'm using grunt to watch and compile all SCSS and JS changes. There's some extra code in there to send out growl notifications and livereload compatability too. Foundation4 is baked in, as it's my preferred frontend framework.
